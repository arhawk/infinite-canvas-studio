# 组件任务拆分文档

## 任务 1.1 Mind Map Branch

### 目标

让 mind map 支持真正的树状分支结构，而不是只靠手动摆放节点。

### 需要实现

- 选中节点后可新增子节点
- 支持折叠 / 展开分支
- 支持快捷键：
- `Tab` 新增子节点
- `Enter` 新增同级节点

### 需要保存的数据

- `parentId`
- `childOrder`
- `collapsed`

### 开发边界

- 折叠只是隐藏子树，不是删除
- 折叠后相关子节点和分支连线都要一起隐藏
- 必须支持 undo/redo
- 必须支持 save/load

### 完成标准

- 可以新增子节点和同级节点
- 可以折叠 / 展开
- undo/redo 正常
- 导出再导入后树结构不丢

## 任务 1.2 Mind Map 小地图

### 目标

在左上角或左下角显示当前整张知识图的缩略视图，帮助用户知道自己在全局中的位置。

### 需要实现

- 显示小地图
- 显示当前主视口范围
- 点击小地图后主视口跳转

### 开发边界

- 小地图只做视图辅助，第一版不需要写入文档
- 需要监听视口变化与节点变化
- 注意性能，不要因为小地图让主画布明显变卡

### 完成标准

- 小地图能显示整图概览
- 主画布缩放 / 平移时小地图同步
- 点击小地图能跳转到对应区域

## 任务 1.3 Mind Map Node 内容映射

### 目标

让节点不仅是一个框，而是承载术语、定义、说明和附件的知识单元。

### 需要实现

- 支持术语 / 词组与定义的一对一映射
- 节点支持附件：
- `file`
- `video`
- `url`
- 点击术语可看到解释 / 定义
- 点击节点可看到它和其他知识点的关系说明

### 建议字段

- `title`
- `body`
- `term`
- `definition`
- `attachments`
- `nodeKind`

### 附件结构建议

```js
{
  id,
  type, // file | video | url
  label,
  value
}
```

### 开发边界

- 第一版定义直接挂在节点上即可
- 第一版“关系展示”先做基础版：
- 列出关系
- 高亮相关节点
- 不需要第一版就做复杂的数据流动画

### 完成标准

- 节点可编辑术语、定义、附件
- 点击节点能看到对应内容
- undo/redo 正常
- save/load 正常

## 任务 1.5 Node 播放功能

### 目标

让教师可以按知识结构顺序播放节点内容。

### 需要实现

- 支持播放模式
- 支持按 DFS 顺序遍历节点
- 支持键盘切换上下一个节点

### 开发边界

- 第一版只支持树状父子关系，不处理所有自由连线
- 第一版键位建议：
- `←` 上一个
- `→` 下一个
- 默认 DFS
- BFS 可作为后续扩展

### 完成标准

- 可进入播放模式
- 可按顺序切换节点
- 视口会自动聚焦当前播放节点

## 任务 1.6 Treemap 多级视图

### 目标

把 mind map 的树结构切换成 treemap 总览视图。

### 需要实现

- 支持从 mind map 切换到 treemap
- 支持点击某个区域进入子层级
- 支持返回上一级

### 开发边界

- treemap 是另一种视图，不是普通节点组件
- 数据源仍然来自 mind map tree
- 切换视图时不要破坏原始画布状态

### 完成标准

- 可以切换 treemap 视图
- 可以 drill down 查看子层级
- 可以返回原画布视图

---

## 任务 1.7 Outline 与 Mind Map 切换

### 目标

支持大纲视图与 mind map 视图互相转换。

### 需要实现

- outline 输入生成树结构
- 现有 tree 结构可导出为 outline
- 支持 outline / mind map 之间切换

### 开发边界

- 必须只有一个主数据源
- 建议 tree 结构作为主数据源
- outline 只是另一种编辑方式，不要变成第二套独立数据

### 完成标准

- outline 可生成 mind map
- mind map 可转成 outline
- 切换前后内容不丢

## 任务 2.1 Binary Calculator

### 目标

增加一个二进制 / 多进制计算工具面板。

### 需要实现

- 2 / 8 / 10 / 16 进制转换
- 基础运算：
- `+`
- `-`
- `*`
- `/`
- 逻辑运算：
- `AND`
- `OR`
- `XOR`

### 开发边界

- 这是工具面板，不建议做成画布节点
- 计算逻辑最好单独抽离，不要全部塞 UI 代码里

### 完成标准

- 转换结果正确
- 运算结果正确
- 输入错误时有明确提示
