Infinite Canvas Studio User Guide / Product Manual

This guide focuses on normal product usage: how teachers build boards, present lessons, share sessions with students, save work, and recover from common limitations. System installation and handover details are covered in the repository README.

Normal Usage Patterns

Infinite Canvas Studio is designed for teachers and students who need a flexible visual workspace instead of a fixed linear slide deck.

Lesson presentationCreate one Page per topic, connect pages with Buttons or connections, then switch to Present mode.
Classroom activityUse Ranking Box, Sticky Notes, shapes, drawing tools, Timer, Emoji Reactions, and Page Compare for discussion and comparison tasks.
Resource boardPlace web pages, videos, images, code examples, attachments, and notes in one navigable canvas.
Shared viewingCreate a temporary room so viewers can follow the teacher camera or browse freely.

Typical Flow

  1. Open the app and stay in Edit mode.
  2. Add Pages and arrange them on the infinite canvas.
  3. Add teaching content with Text, Sticky Note, Image, Iframe, JS Code Runner, Local Video, Ranking Box, and Shapes.
  4. Use the Outline and connections to build a meaningful lesson structure.
  5. Save a JSON or HTML backup before presenting.
  6. Switch to Present mode and deliver the lesson.
  7. Share through a room link if students need to join remotely.

Teacher Edit Workflow

Build The Canvas

  1. Click Components on the left toolbar and drag a component to the canvas.
  2. Use Page as the main lesson surface. Place related components on top of the Page.
  3. Use Text and Sticky Note for explanations, questions, reminders, and discussion notes.
  4. Use Image, Iframe, Local Video, and JS Code Runner for media and demonstrations.
  5. Use Shape and Button style controls to adjust fill, opacity, border, text size, and layer order.

Organize Content

  • Use the right Outline to list important Pages or components in teaching order.
  • Use connections to show relationships between components.
  • Use Buttons for explicit click-to-jump navigation in Present mode.
  • Use attachments/bookmarks on attachment-capable components when external files or URLs should remain close to a Page.

Presentation Workflow

  1. Prepare and save the board in Edit mode.
  2. Switch the mode capsule from Edit to Present.
  3. Use Buttons, connected Pages, arrow navigation, and saved focus positions to move through the lesson.
  4. Move the mouse to the top of the board to reveal the presentation toolbar.
  5. Use Timer, Binary Calculator, Emoji Reactions, drawing tools, and Page Compare when relevant to the lesson.
  6. Press Esc to leave Present mode and return to Edit mode.

Student / Viewer Sharing Workflow

Infinite Canvas Studio supports temporary online viewing sessions through the Share button. The final product uses a Host / Viewer room model rather than multi-user board editing.

Teacher modeShare resultTypical use
EditRoom linkThe teacher can prepare or adjust the board while the Host broadcasts board state and camera updates.
PresentRoom linkThe teacher presents and viewers follow or browse without editing the host state.

Create A Room Link

  1. Click Share in the top toolbar.
  2. Optionally enter a room password.
  3. Click Create room.
  4. Share the displayed URL or QR code with viewers.

Viewer Modes

  • Host: follow the teacher's camera position and zoom.
  • Viewer: pan and zoom freely without forcing the teacher camera.
  • If a viewer pans or zooms while following Host, Infinite Canvas Studio switches that viewer to free Viewer mode.
Local file:// HTML exports hide Share because local files cannot reliably use the online room relay.

Save / Load / Export Workflow

Use the top toolbar save/load buttons or keyboard shortcuts Mod+S and Mod+O.

FormatBest forNotes
Save as JSONEditable backup and later restoreSaves components, styles, outline, connections, drawings, attachments metadata, background state, and viewport state.
Save as HTMLOffline viewing, submission, or sharing a self-contained snapshotEmbeds the board state into a single browser-openable file.
Save as PROJProject-folder handover with attachmentsRequires a Chromium-based browser with File System Access API support.
Load HTML/JSONRestore a saved boardLoading replaces the current board after confirmation.
Load PROJRestore a project folderRequires Chromium-based browser support.

Recommended Backup Habit

  1. Save JSON while actively editing.
  2. Export HTML before a demo, class, or submission.
  3. Use PROJ when local attachments need to remain bundled with the board.

FAQ And Common Limitations

Which export format should I use?

Use JSON for continued editing, HTML for offline viewing or submission, and PROJ when a Chromium-based browser can export a folder with attachments.

Why can't some websites display inside an Iframe?

Some websites block embedding through browser security policies. Use a plain link or attachment when embedding is blocked.

Why does a local exported HTML file hide Share?

Local file:// pages cannot reliably connect to the online room relay. Use the hosted app for live room sharing.

Can viewers edit the host board?

No. Room viewers cannot edit, undo/redo, or load a document over the host state. They can follow the Host camera, switch to free Viewer camera, react with Emoji Reactions, and save/export their own copy.

Can a Button jump both ways automatically?

A Button is a one-way jump. Create one Button in each direction when two-way navigation is needed.

Where is installation and recovery documentation?

System requirements, installation, error recovery, backup/restore, and end-of-life notes are in the repository README.
