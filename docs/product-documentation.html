Product Overview

Mimi is an infinite-canvas visual organization and presentation tool. Users can create pages, text blocks, sticky notes, images, embedded web pages, code runners, local videos, ranking activities, and shapes, then organize them with an outline, connections, jump buttons, and presentation controls.

Lesson presentationsOrganize multiple Pages into a non-linear lesson and control the path with buttons and connections.
Classroom interactionUse Ranking Box, pens, highlighter, timer, and Page Compare for activities.
Resource navigationPlace web pages, images, videos, attachments, and explanatory text in one board.
Offline deliveryExport JSON for continued editing or single-file HTML for browser viewing.

Create Your First Presentation Board

  1. Enter Edit mode.
  2. Click Components in the left toolbar and choose Page.
  3. Add Text, Sticky Note, Image, Iframe, JS Code Runner, or Local Video.
  4. Create a Button and connect it to a target Page or component.
  5. Add key components to the right-side Outline.
  6. Switch to Present and test page navigation, button jumps, and Page Compare.
  7. Export JSON or HTML.
Open the local development version 
pnpm install
pnpm dev

Visit http://localhost:3000.

How to use Page

A Page works like a slide. Put related text, images, buttons, and videos on the same Page so they can move together as one presentation unit.

How to use Button

A Button is a one-way jump. For two-way navigation, create one Button on each side and point each Button to the other destination.

Edit Mode

Edit mode is for creating and organizing content. It supports selection, multi-selection, marquee selection, movement, resizing, style editing, drawing, connections, outline management, and export.

Present Mode

Present mode is for delivery. Editing panels collapse, the board area is maximized, and users can navigate with page-edge buttons, Buttons, connection controls, Page Compare, and fullscreen display.

Online room viewers are read-only. They cannot enter Edit mode or load a document that replaces the host state.

Shapes

Shapes include rectangle, circle, diamond, and triangle. They support text, font size, text color, fill, opacity, stroke color, stroke width, layer ordering, and connections.

Iframe limitation

Some websites block embedding through security policies. If a site cannot display, save it as an attachment or text link instead.

Outline, Connections, And Navigation

Outline

The right-side Outline structures the freeform canvas. Select components and click Add selected, or drag components into the outline. Click an item to center the viewport, double-click to rename it, and drag or use direction controls to adjust order and hierarchy.

Connections

Connections express relationships between components and can also support presentation navigation. Use connections for conceptual relationships and Button for explicit click-to-jump actions.

Overview

Overview is the bottom-right minimap. It shows the current viewport, lets users click to move around, marks selected components, and warns about Pages without connection targets.

Page Compare

In Present mode, select one Page, hold Shift, and click another Page to compare them. The comparison window supports swapping, zooming, fit-to-window, fullscreen, and Esc close.

Save, Export, And Online Sharing

JSONBest for continued editing. Saves components, styles, connections, outline, attachments, drawings, annotations, and viewport state.
HTMLBest for offline delivery or submission. Produces one browser-openable snapshot file.
LoadLoading JSON replaces the current board and resets the undo/redo baseline.
ShareCreate a four-digit room link and QR code, with an optional password.

Online rooms

A host creates a room and shares the link. Viewers can follow the host camera or switch to free viewing. Viewers can save their own copy but cannot edit the host board.

Local file:// HTML exports hide Share because local files cannot reliably use the online room service.

Developer Appendix

This section is not the main user manual. It provides the minimum path for people who need to start the project from scratch.

Requirements

  • Node.js ^20.19.0 or >=22.12.0
  • pnpm 10.23.0

Local development

pnpm install
pnpm dev

Room relay server

pnpm server

Build and test

pnpm build
pnpm test:unit
pnpm test:e2e
pnpm test

Secondary development notes

  • Components live in src/component/; behaviors live in src/plugins/.
  • Components own creation, serialization, and restoration.
  • Reversible changes should emit node:change:start and node:changed.
  • New components need at least one undo/redo or document-load roundtrip test.
  • Normal frontend feature work should not change server/ unless backend changes are explicitly required.
