# Mimi Mind Map Infinite Canvas 产品文档

版本对应：当前仓库前端应用版本  
适用对象：教师、学生、演示者、内容创作者，以及需要在无限画布上组织课程、活动或展示材料的用户

## 1. Documentation、README 和用户手册的区别

在这个项目里，建议把三类文档分开理解：

| 类型 | 主要读者 | 应该回答的问题 | 推荐位置 |
| --- | --- | --- | --- |
| README | 开发者、评审者、部署者 | 这个项目是什么？如何安装、运行、测试？代码结构在哪里？ | `README.md` |
| Product Documentation | 最终用户、课程展示对象、评审者 | 如何创建第一个作品？每个功能怎么用？适合哪些使用场景？ | `docs/product-documentation.md` |
| Developer Guide | 二次开发者 | 如何扩展组件、插件、历史记录、序列化、测试？ | 可放在 `AGENTS.md`、`docs/developer-guide.md` 或 README 的开发章节 |

因此，“如何创建第一个项目”和“如何使用各个功能”完全可以放在同一份产品 documentation 里。更自然的结构是：先用一个完整场景带用户完成第一份作品，再按功能模块做参考说明。

## 2. 产品概述

Mimi 是一个基于无限画布的可视化内容组织与展示工具。用户可以在画布上创建页面、文本、便签、图片、网页嵌入、代码运行器、本地视频、排序活动和图形，并通过目录、连接线、跳转按钮、演示模式和页面对比功能，把零散素材组织成可讲解、可展示、可互动的内容空间。

它适合：

- 制作课程讲解画布或课堂活动材料
- 创建非线性的演示文稿
- 组织 mind map、概念图、流程图和资料导航
- 在同一空间里混合网页、图片、视频、代码和文本说明
- 面向观众进行只读展示，或通过在线房间分享当前画布状态

核心特点：

- 无限画布：支持平移、缩放和全局概览
- 编辑/演示双模式：编辑时组织内容，演示时最大化观看区域
- 组件化内容：页面、按钮、文本、便签、图片、Iframe、排序盒、JS 代码运行器、本地视频
- 图形与连接：矩形、圆形、菱形、三角形、组件连接线和跳转导航
- 本地保存：支持 JSON 文档保存/加载，以及可离线打开的单文件 HTML 导出
- 课堂辅助：画笔、高亮、橡皮擦、页面对比、计时器、二进制计算器、附件书签、目录折叠

## 3. 快速开始：创建你的第一个演示画布

### 3.1 打开应用

开发环境下，进入仓库根目录后运行：

```bash
pnpm install
pnpm dev
```

浏览器打开：

```text
http://localhost:3000
```

如果只是作为最终用户使用，请打开部署后的网页或导出的单文件 HTML。

### 3.2 创建第一个 Page

1. 保持在 `Edit` 模式。
2. 点击左侧工具栏的 `Components`。
3. 选择 `Page`，页面会被添加到当前视图中心；也可以拖拽组件到画布指定位置。
4. 选中 Page 后，可通过页面工具栏调整背景、文字样式、附件和层级。

Page 可以理解为传统幻灯片的一页。放在 Page 上的子组件可以跟随 Page 一起移动，适合把一页课件、一个活动步骤或一个展示单元组织在一起。

### 3.3 添加内容组件

在 `Components` 中继续添加：

- `Text`：放置标题、说明文字、问题文本
- `Sticky Note`：放置提示、课堂讨论点、临时说明
- `Image`：上传图片素材
- `Iframe`：嵌入网页
- `JS Code Runner`：展示和运行 JavaScript 示例
- `Local Video`：导入并播放本地视频
- `Ranking Box`：创建排序活动
- `Button`：创建跳转按钮

双击可编辑内容的组件，或在选中后使用对应的浮动工具栏进行样式调整。

### 3.4 建立跳转和连接

1. 选中需要建立关系的组件。
2. 使用 `Connect to` 相关操作把它连接到目标组件。
3. 对于 `Button`，连接目标后，在 `Present` 模式下点击按钮即可跳转到目标。

如果希望两个页面可以双向跳转，需要分别在两个页面中创建按钮，并让它们分别指向对方。

### 3.5 加入目录 Outline

右侧 `Outline` 面板用于创建可导航的结构目录：

1. 选中一个或多个组件。
2. 点击 `Add selected`，或拖拽选中组件加入目录。
3. 点击目录项可以让视图居中到对应组件。
4. 拖拽目录项或使用方向按钮可以调整层级和顺序。
5. 双击目录项可以重命名，并同步更新画布上的对应内容。
6. 折叠目录分支时，画布上的子节点也会同步折叠隐藏。

### 3.6 切换到 Present 模式

完成编辑后，切换到 `Present`：

- 左右编辑面板会自动收起，让画布区域最大化
- 鼠标移到顶部区域可以呼出模式切换和演示工具
- 可以隐藏或显示所有笔迹
- 可以进入全屏展示
- Page 放大后，视口边缘会出现导航浮球，用于在页面之间切换
- 连接线边缘也可以出现跳转入口，帮助观众沿关系路径浏览

## 4. 界面总览

| 区域 | 用途 |
| --- | --- |
| 左侧工具栏 | 切换 Cursor、Brush tools、Eraser、Shapes、Components，并打开计时器、计算器、背景样式和全局适配 |
| 顶部栏 | 显示项目标题、保存/加载/分享、Edit/Present 模式切换，以及上下文工具 |

| 浮动面板 | 根据选中组件显示对应样式、附件、连接、视频、代码等控制 |

## 5. 模式说明

### 5.1 Edit 模式

Edit 是内容创建和组织模式。此模式下可以：

- 创建、移动、缩放、旋转组件
- 单选、多选和框选
- 编辑文本、便签、按钮、页面和图形样式
- 使用画笔、高亮、注释和橡皮擦
- 建立组件连接
- 加入目录并调整目录层级
- 保存、加载、导出和分享

常见选择方式：

- 单击组件：单选
- `Ctrl`/`Cmd` + 单击：多选
- `Shift` + 拖拽空白区域：框选
- 选中后拖拽：移动组件
- 拖拽空白区域：平移画布

### 5.2 Present 模式

Present 是展示模式。此模式下重点是观看、讲解和导航：

- 编辑面板自动收起
- 用户可以平移和缩放查看内容
- 可以用按钮、页面边缘浮球、连接线导航入口跳转
- 可以显示或隐藏绘制笔迹
- 可以打开页面对比
- 在线房间中的 viewer 会被限制为只读观看

## 6. 工具栏功能

### 6.1 Cursor

Cursor 用于选择、移动和整理内容：

- 点击选择组件
- 拖拽移动组件
- 多选后可以整体移动
- 在空白区域拖拽可移动画布

### 6.2 Brush tools

Brush tools 包含：

- Pen：常规书写
- Pencil：带轻微手绘效果的笔迹
- Highlighter：半透明高亮笔

每类笔刷有预设颜色、宽度和透明度。它们适合课堂讲解、临时标注、重点圈画和演示时补充说明。

### 6.3 Eraser

Eraser 用于删除整条笔迹，而不是只擦除笔迹的一小段。可以调整橡皮擦大小，也可以清空所有笔迹。

### 6.4 Shapes

Shapes 用于创建基础图形：

- Rectangle
- Circle
- Diamond
- Triangle

图形支持：

- 修改文字
- 修改字体大小和文字颜色
- 修改填充颜色、透明度、边框颜色和边框宽度
- 调整上下层级
- 连接到其他组件

### 6.5 Components

Components 是主要内容组件入口。点击后会打开组件列表，选择或拖拽即可创建组件。

## 7. 组件说明

### 7.1 Page

Page 是基础展示页面，可以把它理解为一张幻灯片或一个内容容器。

常见用途：

- 一页课件
- 一个讨论问题
- 一个活动步骤
- 一组相关素材的承载区域

支持功能：

- 修改背景和样式
- 添加文本、图形、图片等子组件
- 添加附件或文件夹
- 与其他组件建立连接
- 调整层级
- 在演示模式下作为导航目标

### 7.2 Button

Button 是跳转按钮，适合制作非线性演示路径。

支持功能：

- 设置按钮文字
- 修改按钮形状、底色、边框、透明度和字体样式
- 连接到目标组件
- Edit 模式下显示指向目标的连接提示
- Present 模式下点击后跳转到目标

注意：Button 默认是单向跳转。要实现双向跳转，需要在两个目标之间分别创建按钮。

### 7.3 Text

Text 是基础文本框，适合标题、说明、问题和注释。

支持功能：

- 编辑文本内容
- 修改字体大小、颜色和基础样式
- 调整大小和位置
- 调整上下层级
- 参与目录和连接

### 7.4 Sticky Note

Sticky Note 是带背景色的文本便签，适合：

- 课堂提示
- 临时想法
- 分组讨论记录
- 重点提醒

支持修改字体、文字颜色、便签底色、大小和层级。

### 7.5 Image

Image 用于上传图片文件。浏览器支持的常见图片格式都可以使用，例如 PNG、JPG、WebP、GIF 等。

支持功能：

- 上传或替换图片
- 调整大小和位置
- 连接到其他组件
- 调整上下层级
- 随文档保存为内嵌数据，便于导出后重新打开

### 7.6 Iframe

Iframe 用于嵌入网页链接。它可以把网页作为画布中的一个组件展示出来。

支持功能：

- 输入 URL
- 在组件区域内查看网页
- 切换交互/展示状态：交互状态下可以直接操作网页内容，展示状态下更适合移动和编排
- 连接到其他组件

限制说明：某些网站会因为自身安全策略禁止被嵌入到 Iframe 中，这不是应用本身的问题。

### 7.7 Ranking Box

Ranking Box 用于排序活动。它适合课堂互动，例如：

- 给步骤排序
- 对观点进行优先级排序
- 做流程重排练习

可以把文本项放入 Ranking Box 中进行排列。放入后文本项主要作为排序对象使用，不适合再当作普通文本框持续编辑。

### 7.8 JS Code Runner

JS Code Runner 用于编写和运行 JavaScript 示例。

主要区域：

- Code：输入 JavaScript 代码
- Preview：显示运行结果或可视化输出
- Console：显示日志和错误信息

常用操作：

- Run：运行代码
- Clear：清空输出
- 调整组件大小以获得更大的代码或预览空间

它适合编程教学、算法演示、课堂实验和交互式示例。

### 7.9 Local Video

Local Video 用于导入本地视频文件并在画布上播放。

支持浏览器视频播放器的常见能力：

- 播放/暂停
- 音量调节
- 全屏
- 播放速度
- 画中画
- 下载或浏览器内置控制

视频文件可以随文档保存为内嵌数据，便于在导出的文档中重新打开。

## 8. 目录 Outline

Outline 是右侧目录面板，用于把自由画布组织成结构化导航。

它可以帮助用户：

- 快速知道当前画布有哪些关键对象
- 在大量内容中快速跳转
- 表达父子层级关系
- 折叠某个分支，让画布更清爽

常用操作：

- 选中组件后点击 `Add selected` 加入目录
- 拖拽目录项改变顺序或层级
- 使用方向按钮微调目录结构
- 双击目录项重命名
- 点击目录项跳转到对应画布组件
- 折叠目录分支，同步隐藏画布上的子组件

右上角会显示当前目录中的项目数量。

## 9. 连接与导航

连接线用于表达组件之间的关系。它既可以作为概念关系，也可以作为演示导航路径。

常见用途：

- 表达页面之间的顺序
- 表达概念之间的因果或依赖
- 把按钮连接到跳转目标
- 在演示模式下沿连接路径移动视角

建议：

- 用连接线表达内容关系
- 用 Button 表达明确的观众点击跳转
- 对重要 Page 尽量配置连接，否则 Overview 会提示未连接页面

## 10. 附件与书签

支持附件的组件，尤其是 Page，可以添加 URL、单个文件或文件夹。

适合场景：

- 给某一页课程添加参考资料
- 给活动页添加素材包
- 给演示页添加外部链接
- 在 Present 模式下像书签一样打开相关文件

在 Present 模式中，Page 的附件会以书签样式显示，点击即可打开对应内容。单个附件可以删除。

## 11. Overview 小地图

右下角 `Overview` 是画布小地图。

它的作用：

- 显示当前视口在整体画布中的位置
- 点击小地图快速移动视角
- 选中组件时显示定位提示
- 折叠或展开小地图
- 提示未连接的 Page，并可逐个跳转检查

左侧工具栏中的全局适配按钮也会出现在 Overview 顶部，用于快速把所有内容适配进视口。

## 12. Page Compare 页面对比

Page Compare 用于在 Present 模式下同时比较两个页面。

使用方式：

1. 切换到 `Present`。
2. 点击选择一个 Page。
3. 按住 `Shift` 再点击另一个 Page。
4. 打开对比窗口。

对比窗口支持：

- 左右并排查看两个页面
- 交换两个页面位置
- 分别缩放页面
- 一键恢复适配窗口大小
- 全屏展示
- `Esc` 关闭对比窗口

它适合对比两种方案、两组答案、课前/课后状态、不同学生作品或不同实验结果。

## 13. 保存、加载与导出

### 13.1 JSON 保存

JSON 是项目的结构化文档格式，适合继续编辑。

保存内容包括：

- 所有组件
- 组件位置、大小、样式和内容
- Page、目录、连接、附件、注释和绘图
- 当前视口位置和缩放

### 13.2 JSON 加载

加载 JSON 会替换当前画布内容。应用会在已有内容时提示确认。

加载完成后，当前文档会成为新的历史记录基线，避免旧的 undo/redo 状态影响新文档。

### 13.3 单文件 HTML 导出

HTML 导出会生成一个可在浏览器中打开的独立文件。它适合：

- 提交作业或课程项目
- 离线展示
- 分享给不需要安装开发环境的人
- 保留当前画布快照

注意：本地 `file://` 打开的 HTML 导出版会隐藏在线分享按钮，因为本地文件不能稳定连接在线房间服务。

## 14. 在线房间分享

在线分享可以让 host 创建一个四位房间号链接，例如 `/room/1234`。观众打开链接后进入 viewer 模式。

Host 可以：

- 创建房间
- 设置可选密码
- 获取分享链接和二维码
- 编辑画布并向 viewer 广播状态
- 广播自己的视角

Viewer 可以：

- 通过链接进入房间
- 输入密码加入受保护房间
- 跟随 host 视角
- 切换到自由观看视角
- 保存当前 JSON 或 HTML 副本

Viewer 不能：

- 进入 Edit 模式
- 修改画布内容
- 加载文档覆盖 host 状态

如果 viewer 在跟随 host 时主动平移或缩放，会自动切换到自由观看视角。

## 15. 课堂辅助工具

### 15.1 Timer

Timer/Stopwatch 可用于课堂限时活动、展示倒计时或练习计时。可以开始、暂停、重置，并设置分钟和秒数。

### 15.2 Binary Calculator

Binary Calculator 是浮动二进制计算器，适合计算机基础、数制转换和课堂快速演示。

### 15.3 Background Style

Style 面板可调整画布背景：

- Default / Colorful 风格
- Blank / Grid / Dot 背景类型
- 预设背景色
- 自定义背景色
- 背景透明度

## 16. 常用快捷键

| 快捷键 | 功能 |

## 17. 推荐工作流

### 17.1 制作课程演示

1. 用 Page 划分每个教学环节。
2. 在 Page 中加入 Text、Sticky Note、Image 和 Video。
3. 用 Button 设计非线性跳转。
4. 用连接线表达页面关系。
5. 把关键 Page 加入 Outline。
6. 在 Present 模式中测试导航。
7. 导出 HTML 用于离线展示，或导出 JSON 方便继续编辑。

### 17.2 制作课堂排序活动

1. 创建 Page 作为活动说明页。
2. 添加 Text 说明任务要求。
3. 添加多个 Text 作为排序项。
4. 添加 Ranking Box。
5. 让学生把文本项放入 Ranking Box 中排序。
6. 使用 Page Compare 对比不同排序结果。

### 17.3 制作编程教学示例

1. 创建 Page。
2. 添加 Text 说明概念。
3. 添加 JS Code Runner。
4. 在代码区输入示例代码。
5. 运行并观察 Preview 或 Console。
6. 用笔刷标注关键输出或错误位置。

## 18. 常见问题

### 为什么有的网站不能在 Iframe 中显示？

部分网站会通过安全响应头禁止被其他网页嵌入。遇到这种情况，可以改为添加链接附件，或换用允许嵌入的页面。

### 为什么导出的本地 HTML 没有 Share 按钮？

本地 `file://` 文件无法稳定使用在线房间服务，因此导出版会隐藏 Share。需要在线分享时，请使用部署后的网页版本。

### JSON 和 HTML 导出该选哪个？

- 想继续编辑：选 JSON。
- 想离线展示或提交一个浏览器可打开文件：选 HTML。
- 想保留项目源数据并可长期维护：JSON 和 HTML 都保留。

### Present 模式下还能编辑吗？

Present 模式主要用于展示，编辑功能会被隐藏或禁用。需要修改内容时请切回 Edit。

### Button 为什么只能跳过去，不能自动跳回来？

Button 表达的是单向跳转。需要双向导航时，需要创建两个 Button，分别指向对方。

## 19. 开发者运行附录

如果文档读者需要从 0 到 1 启动项目，请参考以下最小流程。

### 19.1 环境要求

- Node.js `^20.19.0` 或 `>=22.12.0`
- pnpm `10.23.0`

### 19.2 本地运行

```bash
pnpm install
pnpm dev
```

访问：

```text
http://localhost:3000
```

### 19.3 房间服务

```bash
pnpm server
```

默认前端房间后端主机是 `au.baitian.moe:3001`。测试可以通过 `window.__ROOM_BACKEND_HOST__` 覆盖。

### 19.4 构建与测试

```bash
pnpm build
pnpm test:unit
pnpm test:e2e
pnpm test
```

第一次运行 Playwright E2E 前可能需要：

```bash
pnpm exec playwright install chromium
```

### 19.5 二次开发提醒

- 新组件应放在 `src/component/`
- 组件负责自己的创建、序列化和恢复
- 行为应放在 `src/plugins/`
- 可撤销修改应发出 `node:change:start` 和 `node:changed`
- 新组件至少要通过 undo/redo 或 document load 的 roundtrip 测试
- 普通功能开发不要修改 `server/`，除非明确需要后端变化
